Unlike `sessionStorage` (used correctly for the JWT token), `localStorage` persists indefinitely — surviving browser restarts — and wasn't cleared on logout, leaving plaintext vault data accessible on a shared or stolen device after sign-out. **Decision:** clear `ee_v3` on logout (simplest option; keeps offline support while logged in, removes the persistence risk after sign-out). Implemented as a single line in `doLogout()` in both `index.html` and `frontend/index.html`. New Playwright test in `tests/frontend/login.spec.js` confirms the cache exists before logout and is gone after. Note this is independent of F04: encrypting vault content in MongoDB doesn't and shouldn't touch this, since the browser legitimately needs plaintext to render the UI. |
